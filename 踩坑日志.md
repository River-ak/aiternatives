## 2026-07-07

- **[路径]** `01:28` — 工作目录名 `mkdir aiternatives` 包含空格，直接 `cd /Users/ak/Workbuddy/mkdir aiternatives && ...` 会被 zsh 拆分成两个 token 导致 `cd: no such file or directory`。解决：路径用双引号包裹 `cd "/Users/ak/Workbuddy/mkdir aiternatives"`。
- **[依赖]** `00:07` — `@astrojs/tailwind@6.0.2` 仅支持 Tailwind CSS v3，若使用 v4 会在 `npm install` 阶段报 `ERESOLVE unable to resolve dependency tree`。解决：移除 `@astrojs/tailwind`，改用 `@tailwindcss/vite` 插件。
- **[字体]** `00:20` — 在 Tailwind v4 中通过 `@import url('...')` 引入 Google Fonts 会触发构建警告 "@import rules must precede all rules"。解决：将字体加载改为 `<link>` 标签放在 `<head>` 中（性能也更好）。
- **[类型]** `01:27` — `astro check` 报错 `Property 'key' does not exist on type 'HTMLAttributes'`。Astro 模板中 `<tr>`、`<span>` 等原生 HTML 元素不支持 React 的 `key` 属性。解决：移除 `key={...}`。
- **[类型]** `01:27` — 测试数据中 `logoUrl: null` 不满足 `logoUrl?: string`（可选但不可 null）。解决：将接口放宽为 `logoUrl?: string | null` 以兼容数据库可能返回的 null。
